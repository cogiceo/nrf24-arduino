6 /// \mainpage NRF24 library for Arduino
7 ///
8 /// This is the Arduino NRF24 library.
9 /// It provides an object-oriented interface for sending and receiving data messages with Nordic nRF24L01
10 /// based radio modules, such as the sparkfun WRL-00691 http://www.sparkfun.com/products/691
11 ///
12 /// The nRF24L01 (http://www.sparkfun.com/datasheets/Wireless/Nordic/nRF24L01P_Product_Specification_1_0.pdf)
13 /// is a low-cost 2.4GHz ISM transceiver module. It supports a number of channel frequencies in the 2.4GHz band
14 /// and a range of data rates.
15 ///
16 /// This library provides functions for sending and receiving messages of up to 32 octets on any
17 /// frequency supported by the nRF24L01, at a selected data rate.
18 ///
19 /// Up to 2 nRF24L01 modules can be connected to an Arduino, permitting the construction of translators
20 /// and frequency changers, etc.
21 ///
22 /// This library provides classes for
23 /// - NRF24: addressed, reliable, retransmitted, acknowledged messages (using nrf24 Enhanced Shockburst)
24 ///
25 /// Example Arduino programs are included to show the main modes of use.
26 ///
27 /// The version of the package that this documentation refers to can be downloaded
28 /// from http://www.airspayce.com/mikem/arduino/NRF24/NRF24-1.11.zip
29 /// You can find the latest version at http://www.airspayce.com/mikem/arduino/NRF24
30 ///
31 /// You can also find online help and disussion at http://groups.google.com/group/NRF24-arduino
32 /// Please use that group for all questions and discussions on this topic.
33 /// Do not contact the author directly, unless it is to discuss commercial licensing.
34 ///
35 /// Tested on Arduino Diecimila, Duemilanove and Mega with arduino 1.0
36 /// on OpenSuSE 12.1 with avr-libc-1.7.1-1.1, cross-avr-binutils-2.19.1-33.1
37 /// and cross-avr-gcc-4.3.3_20100125-28.1, and with arduino-0021 on the same platforms
38 ///
39 /// \par Packet Format
40 ///
41 /// All messages sent and received by this NRF24 library must conform to this packet format, as specified by
42 /// the nRF24L01 product specificaiton:
43 ///
44 /// - 1 octets PREAMBLE
45 /// - 4 octets ADDRESS
46 /// - 9 bits packet control field
47 /// - 0 to 32 octets PAYLOAD
48 /// - 2 octets CRC
49 ///
50 /// \par Connecting nRF24L01 to Arduino
51 ///
52 /// The physical connection between the nRF24L01 and the Arduino require 3.3V, the 3 x SPI pins (SCK, SDI, SDO),
53 /// a Chip Enable pin and a Slave Select pin.
54 /// If you are using the Sparkfun WRL-00691 module, it has a voltage regulator on board and
55 /// can be run with 5V VCC
56 /// The examples below assume the Sparkfun WRL-00691 module
57 ///
58 /// Connect the nRF24L01 to most Arduino's like this (Caution, Arduino Mega has different pins for SPI,
59 /// see below),
60 /// \code
61 /// Arduino Sparkfun WRL-00691
62 /// 3V3 or 5V----VCC (3.3V to 7V in)
63 /// pin D8-----------CE (chip enable in)
64 /// SS pin D10----------CSN (chip select in)
65 /// SCK pin D13----------SCK (SPI clock in)
66 /// MOSI pin D11----------SDI (SPI Data in)
67 /// MISO pin D12----------SDO (SPI data out)
68 /// IRQ (Interrupt output, not connected)
69 /// GND----------GND (ground in)
70 /// \endcode
71 ///
72 /// For an Arduino Leonardo (the SPI pins do not come out on the Digital pins as for normal Arduino, but only
73 /// appear on the ICSP header)
74 /// \code
75 /// Leonardo Sparkfun WRL-00691
76 /// 3V3 or 5V----VCC (3.3V to 7V in)
77 /// pin D8-----------CE (chip enable in)
78 /// SS pin D10----------CSN (chip select in)
79 /// SCK ICSP pin 3----------SCK (SPI clock in)
80 /// MOSI ICSP pin 4----------SDI (SPI Data in)
81 /// MISO ICSP pin 1----------SDO (SPI data out)
82 /// IRQ (Interrupt output, not connected)
83 /// GND----------GND (ground in)
84 /// \endcode
85 /// and initialise the NRF24 object like this to explicitly set the SS pin
86 /// NRF24 nrf24(8, 10);
87 ///
88 /// For an Arduino Mega:
89 /// \code
90 /// Mega Sparkfun WRL-00691
91 /// 3V3 or 5V----VCC (3.3V to 7V in)
92 /// pin D8-----------CE (chip enable in)
93 /// SS pin D53----------CSN (chip select in)
94 /// SCK pin D52----------SCK (SPI clock in)
95 /// MOSI pin D51----------SDI (SPI Data in)
96 /// MISO pin D50----------SDO (SPI data out)
97 /// IRQ (Interrupt output, not connected)
98 /// GND----------GND (ground in)
99 /// \endcode
100 /// and you can then use the default constructor NRF24().
101 /// You can override the default settings for the CSN and CE pins
102 /// in the NRF24() constructor if you wish to connect the slave select CSN to other than the normal one for your
103 /// Arduino (D10 for Diecimila, Uno etc and D53 for Mega)
104 ///
105 /// Caution: on some Arduinos such as the Mega 2560, if you set the slave select pin to be other than the usual SS
106 /// pin (D53 on Mega 2560), you may need to set the usual SS pin to be an output to force the Arduino into SPI
107 /// master mode.
108 ///
109 /// Caution: this module has not been proved to work with Leonardo, at least without level
110 /// shifters between the NRF24 and the Leonardo. Tests seem to indicate that such level shifters would be required
111 /// with Leonardo to make it work.
112 ///
113 /// It is possible to have 2 radios conected to one arduino, provided each radio has its own
114 /// CSN and CE line (SCK, SDI and SDO are common to both radios)
115 ///
116 /// \par Example programs
117 ///
118 /// The following example programs are provided:
119 /// -nrf24_ping_client, nrf24_pin_server. This is a matched pair. The client sends (acknowledged)
120 /// 4 byte timestamp to the server which replies (acknowledged).
121 /// The client measures the round-trip time and prints it. Typical RTT is 1-2 msec.
122 /// These are compatible radio wise with the ping_client and ping_server programs that come with the Mirf library
123 /// though the electrical connections are different
124 /// -nrf24_specan. Example sketch showing how to create a primitive spectrum analyser
125 /// with the NRF24 class. The nRF24L01 received power detector is only one bit, but
126 /// this will show which channels have more than -64dBm present.
127 /// -nrf24_audio_tx, nrf24_audio_rx. This is a matched pair. The clinet sends a stream of audio samples measured
128 /// from analog input 0 to the receiver, which reconstructs them on output D6. See comments in those files for
129 /// electrical requirements. The pair demonstrates the use of NRF24 in NOACK modefor improved performance
130 /// (but no reliability). Can achieve 6.4kHz sample rate. Dont expect good quality audio!
131 ///
132 /// \par Radio Performance
133 ///
134 /// The performance of this radio seems to be very good. I was able to build ping client/server
135 /// that was able to achieve over 800 round trips per second
136 /// (at 0dBm power, 2Mbps, channel 1, 4 byte payload each way, 1 checksum byte) when the radios were next to each other.
137 /// This rate could still be achieved at 15m distance, but the orientation of the radios and
138 /// obstructions became critical. The human body can easily block these signals.
139 /// Best response was when the chip antennas were broadside to each other.
140 ///
141 /// It is possible to get even better streaming performance using NOACK mode (see the nrf24_audio_tx sample)
142 /// at the cost of nop reliability.
143 /// In NOACK mode, at 2Mbps, 32 byte payload, can get about 1900 packets per second: 60800 bytes of payload per second
144 ///
145 /// Frequency accuracy may be debatable. For nominal frequency of 2401.000 MHz (ie channel 1),
146 /// my Yaesu VR-5000 receiver indicated the center frequency for my test radios
147 /// was 2401.121 MHz. Its not clear to me if the Yaesu
148 /// is the source of the error, but I tend to believe it, which would make the nRF24l01 frequency out by 121kHz.
149 ///
150 /// \par Radio operating strategy and defaults
151 ///
152 /// The radio is enabled all the time, and switched between TX and RX modes depending on
153 /// whether there is any data to send. Sending data sets the radio to TX mode.
154 /// After data is sent, the radion automatically returns to Standby II mode. Calling waitAvailable() or
155 /// waitAvailableTimeout() starts the radio in RX mode.
156 ///
157 /// The radio is configured by default to Channel 2, 2Mbps, 0dBm power, 5 bytes address, payload width 1, CRC enabled
158 /// 1 byte CRC, Auto-Ack mode. Enhanced shockburst is used.
159 /// P1 is the receive pipe. P0 is set to the transmit address to enable autoack.
160 ///
161 /// \par Memory
162 ///
163 /// Memory usage of this program is minimal. The compiled ping client and server programs are about 4000 bytes.
164 /// RAM requirements of the library are minimal.
165 ///

387 /// \class NRF24 NRF24.h <NRF24.h>
388 /// \brief Send and receive addressed, reliable, acknowledged datagrams by nRF24L01.
389 ///
390 /// This base class provides basic functions for sending and receiving addressed, reliable,
391 /// automatically acknowledged and retransmitted
392 /// datagrams via nRF24L01 of arbitrary length to 32 octets per packet.
393 /// Sender and receiver must each know the addreesses of the other, so arbitrary meshes and stars are
394 /// not possible at this level.
395 /// Directed replies (ie replies sent back to the original sender) are not possible
396 /// (the address of the sender is not carried in the message).
397 /// See subclasses for support for this.
398 ///
399 /// Subclasses may use this class to implement streams,
400 /// mesh routers, repeaters, translators etc.
401 ///
402 /// On transmission, the addresses of this node defaults to 0x0000000000, unless changed by a subclass.
403 ///
404 /// The radio is configured to use Enhanced Shockburst with retransmits.
405 /// TX_ADDR and RX_ADDR_P0 are set to the transmit address (ie the address of the destination for the next message
406 /// RX_ADDR_P1 is set to the address of this node
407 /// RX_ADDR_P2 is set to RX_ADDR_P1 with the LSbyte set to 0xff, for use as a broadcast address
408 ///
409 /// Naturally, for any 2 radios to communicate that must be configured to use the same frequency and
410 /// data rate, and with compatible addresses
411 class NRF24
412 {
413 public:
